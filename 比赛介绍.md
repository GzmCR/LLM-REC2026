# 快手探索者 LLM-Rec 挑战赛赛题介绍

快手探索者 LLM-Rec 挑战赛是一项围绕“大模型推荐系统”展开的算法竞赛。它关注的核心问题是：**如何让大语言模型不仅会生成文本，还能理解用户兴趣、理解物料内容，并做出更准确的推荐决策**。

传统推荐系统通常依赖用户点击、浏览、点赞、收藏等行为数据，通过协同过滤、深度学习排序模型、序列推荐模型等方法预测用户可能感兴趣的内容。而 LLM-Rec 则进一步引入大语言模型能力，希望模型能够同时具备“懂内容、懂用户、懂推荐、懂世界”的综合能力。也就是说，模型不只是根据历史行为做机械匹配，还要能理解用户兴趣变化、内容语义、推荐场景以及更广泛的知识背景。

本次比赛指定使用 **OneReason-0.8B** 作为基础模型进行迭代，参赛者需要在官方给定的数据和 baseline 基础上，对模型进行训练、微调和优化。根据赛事 FAQ，比赛允许选手基于官方 baseline 代码继续开发，也可以使用万擎平台的模型精调功能；但模型只能基于 OneReason-0.8B 进行迭代，评测时会校验模型 config 是否符合要求。Baseline 不提供 RL 训练代码，若采用强化学习方法，需要参赛者自行实现。

从任务形式看，这不是一个单纯“写代码跑模型”的比赛，而是一个综合型推荐系统挑战。参赛者需要完成数据理解、行为序列建模、文本生成式推荐、模型微调、评测提交和实验复盘等多个环节。官方数据集、模型训练、推理提交和在线评测均在快手万擎平台完成；审核通过后，参赛账号会开通赛事资源权限，选手需要先完成万擎实名认证，才能下载数据、训练模型并查看评测结果。

对入门者来说，可以把这场比赛理解为三个层次：

第一层是**推荐系统问题**。模型需要根据用户历史行为判断用户下一步可能喜欢什么。这部分类似“猜你喜欢”，重点是理解用户兴趣。

第二层是**大模型微调问题**。比赛不是让你从零训练一个大模型，而是在 OneReason-0.8B 这个基础模型上，通过 SFT、参数调整、数据增强、蒸馏或其他策略提升推荐效果。

第三层是**竞赛工程问题**。每天正式评测提交次数有限，排行榜每日固定时间更新，因此参赛者不能盲目试错，而要建立本地验证、实验记录、模型版本管理和提交策略。FAQ 中说明，排行榜正式评测每天最多提交 3 次，每天下午 15 点刷新提交次数，排行榜每日 15:00 更新。

比赛支持 1—3 人组队，面向全球范围内全日制在校学生。每位参赛者只能加入一个队伍，报名截止后队伍名称和成员固定，不能中途新增、删除或替换成员。队长还需要满足万擎平台实名认证要求。

本次比赛的吸引力不仅在于奖金，也在于它非常贴近当前推荐系统和大模型结合的前沿方向。根据 FAQ，初赛结束后有 50 支队伍晋级复赛；奖项包括冠军、亚军、季军、4—10 名奖励以及技术创新专项奖，冠亚季军还可获得快手 K-Star 级别 offer，决赛前 20 名队伍可免笔试直入快手算法终面。

对于刚入门的参赛者，最重要的不是一开始就追求复杂方法，而是先完成四件事：第一，理解推荐系统的基本逻辑；第二，跑通官方 baseline；第三，掌握 OneReason-0.8B 的训练和提交流程；第四，建立一套稳定的实验记录方法。只有先把完整流程跑通，后续的数据处理、SFT 优化、loss 调整、蒸馏或 RL 才有实际意义。

一句话概括：**这场比赛考察的是在统一基础模型和真实推荐场景约束下，如何通过数据、训练策略和推荐理解能力，把一个 0.8B 大模型调教成更懂用户、更懂内容、更会推荐的模型。**
